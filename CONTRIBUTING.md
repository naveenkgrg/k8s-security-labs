# Contributing to Kubernetes Security Labs

Thank you for your interest in contributing! We welcome contributions of all kinds.

## How to Contribute

### Report Issues
- Describe the problem clearly
- Include steps to reproduce
- Mention your Kubernetes version and environment

### Submit Improvements
- Fork the repository
- Create a feature branch: `git checkout -b improve/feature-name`
- Make your changes
- Test your changes
- Commit with clear messages: `git commit -m "Add: clear description"`
- Push to your fork and create a pull request

### Content Guidelines
- Keep instructions clear and practical
- Test all commands before submitting
- Use consistent formatting with existing labs
- Include explanations for complex steps
- Ensure compatibility with recent Kubernetes versions

### Types of Contributions Welcome
- New labs or exercises
- Clarifications and corrections
- Additional examples or use cases
- Better explanations of concepts
- Bug fixes and improvements
- Documentation enhancements

## Code of Conduct

Be respectful, inclusive, and constructive in all interactions.

## Questions?

Feel free to open an issue to discuss ideas before investing significant effort.
